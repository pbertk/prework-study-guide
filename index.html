HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • The Selector (first term before curly brackets) directs which element will be getting styled
  • Using the wildcard (*) for the selector means that all elements will be styled the same way
  • Or, for sub-groupings of impacted selectors, more than one selector can be applicable at the same time by using a comma
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Text-align is used to direct text alignment

Git

  • git status: checks what branch we are currently on
  • git checkout main: switches to the main branch
  • git pull origin main: syncs the local main branch with the main branch on GitHub. Do this before creating a new local working branch as well as before committing working branch back to GitHub repository
  • git checkout -b branch-name: creates a new working branch and switches to it
  • git add -A: used to stage all files in the working branch in prep for commit
  • git commit -m: commits the staged branch to main; follow the -m flag with a descriptive message in quotes ""
  • git restore --staged (file): unstages files
  • git push origin (branch): pushes committed changes in local branch to GitHub repo

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • 'If'/'Else' statements can be nested.
  • The code block in curly brackets will execute if a truthy 'if' statement is encountered, ans script will stop
  • An Array is a variable that has multiple possible contents. Note that in JS, indexing starts at zero
  • A 'for' loop can then be applied to an array, to iterate through all possible array options
  • A function is a set of instructions. They need to be both defined AND called
  • Don't forget to tie the script to the index.html (place just before /body tag).
  • ChromeDev console can be used as a sandbox to test code modifications.
